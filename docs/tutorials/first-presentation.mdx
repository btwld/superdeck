# First Presentation Tutorial

This tutorial walks you through creating a complete presentation from scratch, introducing SuperDeck's key features along the way.

## What We'll Build

A presentation about Flutter development with:
- Title slide with centered content
- Multi-column layouts  
- Images and code examples
- Interactive DartPad embed
- Custom styling

## Prerequisites

- SuperDeck installed and set up ([Getting Started](/getting-started))
- Basic understanding of Markdown

## Step 1: Create the Title Slide

Start with a simple title slide in your `slides.md`:

```markdown
---
title: Flutter Development Essentials
---

@section

@column {
  align: "center"
}

# Flutter Development Essentials

## Building Beautiful Apps with Ease

**Your Name**  
*Today's Date*

---
```

**Build and test:**
```bash
superdeck build --watch
flutter run
```

## Step 4: Add Code Examples

Include syntax-highlighted code:

```markdown
## Your First Flutter App

@section

@column

Here's a simple Flutter app:

```dart
import 'package:flutter/material.dart';

void main() {
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Scaffold(
        appBar: AppBar(title: Text('Hello Flutter')),
        body: Center(
          child: Text('Hello, World!'),
        ),
      ),
    );
  }
}
```

@column

### Key Concepts:
- **Widgets** - Everything is a widget
- **MaterialApp** - App structure
- **Scaffold** - Page layout
- **StatelessWidget** - Immutable widgets

---
```

## Step 5: Interactive Content

Add a DartPad for live coding:

```markdown
## Try It Yourself

@section

@column

- Modify the text
- Change colors
- Add more widgets

  flex: 2
}

@dartpad {
  id: "your_dartpad_id"
  theme: "dark"

**Note:** Replace `your_dartpad_id` with an actual DartPad ID from [dartpad.dev](https://dartpad.dev).

## Tips for Great Presentations

1. **Keep slides focused** - One concept per slide
2. **Use visuals** - Images and code examples enhance understanding  
3. **Test early** - Use watch mode for rapid iteration
4. **Practice navigation** - Test keyboard and touch controls
5. **Consider your audience** - Adjust content complexity accordingly
